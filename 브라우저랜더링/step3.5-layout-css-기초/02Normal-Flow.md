# Normal Flow

## 한 문장으로

Normal Flow는 별도의 위치 지정이나 레이아웃 도구를 쓰지 않았을 때, 브라우저가 요소를 기본 규칙에 따라 배치하는 흐름입니다.

## 왜 Layout 전에 알아야 할까?

브라우저는 CSS가 없어도 HTML 요소를 화면에 배치합니다.

```html
<h1>제목</h1>
<p>첫 번째 문장</p>
<p>두 번째 문장</p>
```

브라우저는 기본적으로 이런 요소들을 위에서 아래로 배치합니다.

```text
제목
첫 번째 문장
두 번째 문장
```

이 기본 배치 흐름을 Normal Flow라고 부릅니다.

Layout 단계는 이 기본 흐름을 바탕으로 요소의 위치와 크기를 계산합니다.

## block 요소의 기본 흐름

`div`, `p`, `h1`, `section` 같은 요소는 기본적으로 block 성격을 가집니다.

block 요소는 보통 한 줄 전체를 차지하고, 다음 요소는 아래로 내려갑니다.

```html
<p>첫 번째</p>
<p>두 번째</p>
```

```text
첫 번째
두 번째
```

첫 번째 요소의 높이가 커지면 두 번째 요소는 더 아래로 밀립니다.

이것이 Layout에서 주변 요소의 영향을 고려해야 하는 이유입니다.

## inline 요소의 기본 흐름

`span`, `a`, `strong` 같은 요소는 기본적으로 inline 성격을 가집니다.

inline 요소는 한 줄 안에서 흐릅니다.

```html
<span>안녕</span>
<span>하세요</span>
```

```text
안녕하세요
```

한 줄에 공간이 부족하면 다음 줄로 줄바꿈됩니다.

## Normal Flow에서 벗어나는 경우

아래 속성들은 요소를 기본 흐름과 다르게 배치하게 만들 수 있습니다.

```css
position: absolute;
position: fixed;
float: left;
display: flex;
display: grid;
```

처음부터 모든 레이아웃 방식을 깊게 알 필요는 없습니다.

하지만 "기본 흐름이 있고, 일부 CSS는 그 흐름을 바꾼다"는 감각은 중요합니다.

## 렌더링과 연결

Layout은 각 요소의 위치를 계산합니다.

그 위치 계산의 출발점이 Normal Flow입니다.

브라우저는 먼저 기본 배치 규칙을 따르고, CSS 속성에 따라 흐름을 바꾸거나 보정합니다.
