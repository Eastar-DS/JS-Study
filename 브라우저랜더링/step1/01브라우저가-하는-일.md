# 브라우저가 하는 일

## 한 문장으로

브라우저는 서버에서 받은 HTML, CSS, JavaScript 같은 파일을 해석해서 사람이 볼 수 있는 화면과 조작 가능한 웹 페이지로 바꾸는 프로그램입니다.

## 왜 이걸 먼저 알아야 할까?

브라우저 렌더링을 공부할 때 처음부터 DOM, CSSOM, Layout 같은 단어를 외우면 흐름이 잘 잡히지 않습니다.

먼저 브라우저가 하는 큰일을 알아야 합니다.

```text
파일 받기 -> 의미 해석하기 -> 화면 계산하기 -> 픽셀로 보여주기 -> 사용자 입력에 반응하기
```

브라우저는 단순히 HTML 파일을 열어주는 뷰어가 아닙니다. HTML, CSS, JavaScript를 함께 해석해서 최종 화면을 계속 갱신하는 실행 환경입니다.

## 예시

아래 HTML을 브라우저가 받았다고 해보겠습니다.

```html
<!doctype html>
<html>
  <head>
    <title>메모 앱</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1>오늘 할 일</h1>
    <button>추가</button>
  </body>
</html>
```

브라우저가 해야 할 일은 많습니다.

```text
1. 이 문자열이 HTML 문법이라는 것을 이해한다.
2. h1, button 같은 요소를 찾는다.
3. style.css 파일이 필요하다는 것을 알아낸다.
4. CSS를 받아서 h1과 button에 어떤 스타일이 적용되는지 계산한다.
5. h1과 button이 화면 어디에 놓일지 계산한다.
6. 글자, 버튼 배경, 테두리 등을 그린다.
7. 사용자가 버튼을 클릭하면 클릭 이벤트를 처리한다.
```

즉, 브라우저는 "파일을 보여주는 도구"가 아니라 "파일을 해석해서 화면과 상호작용을 만드는 도구"입니다.

## 핵심 구분

브라우저가 하는 일을 크게 나누면 다음과 같습니다.

```text
네트워크: HTML, CSS, JS, 이미지 파일을 가져온다.
파싱: 문자열을 의미 있는 구조로 해석한다.
렌더링: 해석한 구조를 화면에 그린다.
실행: JavaScript를 실행하고 사용자 입력에 반응한다.
```

브라우저 렌더링 공부에서는 이 중 특히 `파싱`과 `렌더링`을 많이 다룹니다.

## 흔한 오해

### 오해 1. 브라우저는 HTML을 그대로 보여준다

아닙니다. HTML은 화면이 아니라 문서 구조를 설명하는 문자열입니다.

브라우저는 HTML을 읽고, 구조화하고, 스타일을 붙이고, 위치를 계산한 뒤 화면에 그립니다.

### 오해 2. 렌더링은 한 번만 일어난다

아닙니다. 처음 페이지를 열 때도 렌더링이 일어나고, JavaScript가 DOM을 바꾸거나 CSS가 바뀌거나 화면 크기가 바뀔 때도 다시 렌더링이 일어날 수 있습니다.

3. 렌더링은 왜 처음 한 번만 일어나는 일이 아닐까요?
