# 학습 로그

## 학습 로그 #1

**시간**: 04/04 19:00 ~ 20:00 (약 60분)
**학습 범위**: 이벤트 루프를 이해하기 위한 계획 세우기

### 1. 첫 학습 전략 설계

- 어떤 전략으로 학습하기로 했는가?
    1. AI를 이용해서 이벤트 루프를 이해하기 위한 단계를 계획하고, 각 단계별 키워드 목록을 받아옴.

    2. 순서대로 공부하기 시작하면서 옵시디언에 정리하고, 모르는 키워드가 더 나오면 AI를 이용해 키워드에 대한 자세한 설명과 구체적인 예시, 생각해볼 것을 제공받음.


- 그 전략을 선택한 이유는?
    하나의 키워드를 완벽하게 이해하는 것 보다 큰 주제를 이해하기위해 그때그때 필요한 내용을 알면서 공부하는 것이 더 효율적이라고 판단.

## 학습 로그 #2

**시간**: 04/04 21:00 ~ 22:00 (약 60분)
**학습 범위**: 이벤트 루프 1단계(JS 런타임 환경) + 2단계(이벤트 루프 메커니즘) 키워드 학습 — 스레드, 메모리 공간, JS 엔진/런타임 환경, setTimeout, fetch, Promise, 콜백함수, 실행 컨텍스트, 이벤트 루프, Task Queue, Microtask Queue

### 1. 이번 타임의 학습 전략

- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
계획한 전략대로 실행함. 스레드 키워드에 대한 자세한 설명을 받고, 해당 설명을 읽어보면서 자연스럽게 1단계나 2단계에 계획되어있던 키워드들도 자세한 설명을 받고 공부할 계획을 세웠음. 다만 계획된 키워드 순서대로 찾아볼 수는 없었음. 스레드를 이해하기위해 필요한 개념들도 모르는 상태였기때문에.

- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
스레드 키워드를 공부하기 시작했음. 스레드에 대한 내용을 AI를 이용해 얻었는데 자연스럽게 1단계와 2단계에 계획되어있던 키워드들을 검색하게 되었음.

### 2. 전략 평가

- 효과적이었던 것과 그 이유
AI를 이용해 스레드 키워드를 이해하기위한 내용들을 찾아보게되어서 효율적으로 공부를 할 수 있다고 생각됨. 만약 키워드 하나하나를 마스터하겠다는 생각으로 시작했다면 원하는 시간인 3일 안에 이벤트 루프를 이해한다는 목표를 이룰 수 없을 것 같음.

- 비효과적이었던 것과 그 이유
1. 기초적인 개념이 아예 없다보니 이벤트 루프를 이해하기위해서 각 키워드들을 어느정도까지 이해해야 하는지 선을 정할 수 없음.
2. 1번과 연계되는 내용인데, 각 키워드들을 이해하려면 필요한 내용이 다를텐데, AI를 이용해서 얻는 답변은 대부분 글자수가 비슷해서 이렇게 공부하는게 맞나에 대한 의문이 있음.
3. 기초적인 개념이 없어서 하나의 키워드를 이해하기 위해 많은 부가적인 키워드들을 공부해야하고, 해당 키워드들을 공부하다보면 처음에 세웠던 계획순서대로 진행할 수 없게됨. 가장 효율적인 학습을 위한 피드백이 필요함.

### 3. AI 피드백

- 자신의 학습 전략에 대해 AI에게 피드백을 요청하고, 유용했던 제안 1가지 이상 기록

**A. 설명 테스트 끼워넣기 (인출 연습)**: 키워드 하나를 읽은 후, AI에게 설명을 요청하기 전에 자신이 먼저 1~2문장으로 설명을 시도한다. 그 후 AI 설명과 비교한다. "어디까지 이해했는지"를 스스로 파악할 수 있어서, 각 키워드를 어느 깊이까지 공부할지 선을 정하는 데도 도움이 된다.

**C. 코드 예측 퀴즈 우선 학습 (생산적 실패)**: 키워드 설명을 읽기 전에, 해당 키워드가 관련된 코드 실행 순서 문제를 먼저 풀어본다. 틀려도 괜찮다. 정답을 모른 채 먼저 시도하면, 이후 설명을 읽을 때 "아, 내가 여기서 틀렸구나"라는 강한 학습 신호가 생긴다. AI 설명의 글자 수가 다 비슷해 보이는 문제도 해결된다 — 자신이 틀린 부분에 집중하면 되니까.

### 4. 다음 타임에 바꿀 것

- 유지할 것과 그 이유
AI 키워드 학습, 자연스러운 확장

- 바꿀 것과 그 이유
설명 테스트 추가, 코드 예측 먼저 풀기
