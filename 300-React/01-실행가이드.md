# React 예시 코드 실행 가이드

이 문서의 예시 코드는 가능한 한 별도 프로젝트 세팅 없이 빠르게 실행해볼 수 있도록 작성했습니다.

## 가장 추천하는 방법

### 1. StackBlitz에서 React 템플릿 열기

- 링크: https://stackblitz.com/examples/react
- 추천 이유:
  - 브라우저에서 바로 실행 가능
  - 별도 설치 없음
  - `App.tsx` 하나만 바꿔도 대부분의 예제를 바로 실험 가능

### 2. CodeSandbox에서 React 템플릿 열기

- 링크: https://codesandbox.io/examples/package/React
- 추천 이유:
  - 파일을 여러 개로 나눠 실험하기 편함
  - 링크 공유가 쉬움

### 3. React 공식 문서 안의 샌드박스 활용

- 링크: https://react.dev/learn/installation
- 한글: https://ko.react.dev/learn/installation
- 추천 이유:
  - 아주 작은 예제는 가장 빠르게 실험 가능
  - 문서 예제를 직접 수정해보기에 좋음

## 가장 간단한 실행 순서

1. StackBlitz 또는 CodeSandbox에서 React 템플릿을 연다.
2. 기본으로 열리는 `App.tsx` 내용을 비운다.
3. 각 step 폴더의 `02-example-*.tsx` 내용을 복사해서 `App.tsx`에 붙여넣는다.
4. `export default function ...` 형태가 유지되는지 확인한다.
5. 브라우저 화면과 콘솔 로그를 함께 관찰한다.

## 파일 종류별 실행 방법

### `02-example-*.tsx`

- 대부분 이 형식입니다.
- React 템플릿의 `App.tsx`에 그대로 붙여넣으면 됩니다.

### `02-example-*.ts`

- React UI 렌더링용이 아니라 콘솔 학습용 예제입니다.
- 브라우저 환경의 `main.ts` 또는 별도 `example.ts`에 붙여넣어 콘솔로 확인하면 됩니다.
- 가장 쉬운 방법은 TypeScript Playground 또는 샌드박스의 일반 TS 파일에서 실행하는 것입니다.

## 예시를 붙여넣을 때 주의할 점

- 예제 하나만 넣고 실행합니다.
- 기존 템플릿의 다른 `export default function App()`와 중복되지 않게 합니다.
- 여러 예제를 한 파일에 동시에 넣지 않습니다.
- `alert`, `console.log`, 느린 반복문은 학습용이므로 동작이 거칠 수 있습니다.

## 어떤 도구를 언제 쓰면 좋은가

### StackBlitz를 더 추천하는 경우

- 빠르게 붙여넣고 바로 보기
- 단일 파일 실험
- 렌더링/리렌더링 관찰

### CodeSandbox를 더 추천하는 경우

- 파일을 둘 이상 나눠서 보고 싶을 때
- 코드 링크를 저장/공유하고 싶을 때
- 조금 더 오래 실험할 때

### React 공식 문서 샌드박스를 추천하는 경우

- state snapshot
- conditional rendering
- key 기본 실험
- effect 기본 실험
