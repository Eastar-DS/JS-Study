실험 3. 이미지 Layout Shift

"사용자가 누르려던 버튼이 마지막 순간에 밀린다" — 가장 짜증나는 UX 버그입니다.
아래 두 데모를 새로고침하면서 비교해보세요. 빨간 버튼을 누르려는 시점에 이미지가 도착하면 어떻게 되는지 관찰합니다.

관찰 포인트

  • 네트워크를 Slow 3G로 설정한 뒤 페이지를 새로고침하세요.
  • 나쁜 예: 이미지가 도착하는 순간 빨간 버튼이 아래로 점프합니다.
  • 좋은 예: 이미지가 도착해도 버튼 위치는 그대로입니다.
  • 이게 바로 Cumulative Layout Shift (CLS)입니다. 구글의 Core Web Vitals 지표 중 하나입니다.
  • 핵심: 브라우저가 Layout을 미리 계산할 수 있도록 크기 정보를 미리 제공해야 합니다.

왜 이게 중요한가

  • 사용자는 누르려던 버튼을 잘못 누르게 됩니다 (예: "구매" 자리에 "환불" 버튼이 와서 잘못 누름).
  • SEO에도 영향을 줍니다. Google은 CLS를 검색 순위에 반영합니다.
  • 크기 정보 한 줄을 추가하는 것만으로 해결됩니다.
