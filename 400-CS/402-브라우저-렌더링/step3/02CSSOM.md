# CSSOM

## 한 문장으로

CSSOM은 CSS 규칙을 브라우저가 계산하고 적용할 수 있도록 만든 객체 모델입니다.

## 왜 CSSOM이 필요할까?

DOM이 HTML을 객체 트리로 바꾼 결과라면, CSSOM은 CSS를 객체 구조로 바꾼 결과입니다.

브라우저는 화면을 그릴 때 각 DOM 요소가 어떤 스타일을 가져야 하는지 알아야 합니다.

```html
<body>
  <h1>제목</h1>
  <p>본문</p>
</body>
```

```css
body {
  font-size: 16px;
  color: #222;
}

h1 {
  font-size: 32px;
}
```

브라우저는 이런 질문에 답해야 합니다.

```text
body의 글자 크기는?
h1의 글자 크기는?
p는 color를 직접 지정하지 않았는데 무슨 색인가?
h1은 body의 color를 상속받는가?
```

이 계산을 위해 CSSOM이 필요합니다.

## CSSOM은 DOM과 함께 쓰인다

DOM은 요소의 구조를 알려줍니다.

```text
body
  h1
  p
```

CSSOM은 스타일 규칙을 알려줍니다.

```text
body: font-size 16px, color #222
h1: font-size 32px
```

브라우저는 둘을 합쳐 각 요소의 최종 스타일을 계산합니다.

```text
body: font-size 16px, color #222
h1: font-size 32px, color #222
p: font-size 16px, color #222
```

`h1`은 글자 크기를 직접 지정받았고, 색상은 body에서 상속받았습니다.

## CSSOM이 없으면 생기는 문제

CSSOM 없이 매번 CSS 문자열을 직접 검색한다고 상상해봅시다.

```text
h1 스타일을 알려면 CSS 파일 전체 문자열에서 h1 규칙을 찾는다.
상속되는 값도 다시 찾는다.
우선순위도 매번 다시 계산한다.
```

이 방식은 느리고 복잡합니다.

브라우저는 CSS를 파싱해서 CSSOM으로 만들어두고, DOM 요소와 연결해 최종 스타일을 계산합니다.

## CSS는 렌더링을 막을 수 있다

브라우저는 화면을 그리기 전에 스타일 정보를 알아야 합니다.

HTML은 먼저 일부를 그릴 수 있을 것 같지만, CSS가 아직 도착하지 않았다면 최종 모습이 달라질 수 있습니다.

예를 들어 CSS가 늦게 도착해서 아래 규칙이 나중에 적용된다고 해봅시다.

```css
body {
  display: none;
}
```

브라우저가 CSS 없이 먼저 화면을 그렸다가, CSS 도착 후 모든 것을 숨기면 사용자는 화면이 번쩍 바뀌는 것을 보게 됩니다.

그래서 CSS는 렌더링에 중요한 자원이고, 브라우저는 CSSOM을 만들기 전까지 렌더링을 기다릴 수 있습니다.
