# HTML 파싱

## 한 문장으로

HTML 파싱은 HTML 문자열을 브라우저가 이해할 수 있는 구조로 해석하는 과정입니다.

## 왜 파싱이 필요할까?

브라우저가 서버에서 받는 HTML은 기본적으로 문자열입니다.

```text
<h1>제목</h1><p>본문</p>
```

사람은 이 문자열을 보고 h1과 p가 있다고 이해할 수 있습니다. 하지만 브라우저가 렌더링을 하려면 문자열 상태로는 부족합니다.

브라우저는 아래처럼 의미 있는 단위로 해석해야 합니다.

```text
시작 태그: h1
텍스트: 제목
종료 태그: h1
시작 태그: p
텍스트: 본문
종료 태그: p
```

이렇게 문자열을 규칙에 따라 의미 있는 조각으로 읽고 구조화하는 과정이 파싱입니다.

## 파싱의 결과는 트리 구조가 된다

HTML은 요소 안에 요소가 들어갈 수 있습니다.

```html
<main>
  <section>
    <h1>렌더링</h1>
    <p>브라우저가 화면을 만드는 과정</p>
  </section>
</main>
```

이 구조는 줄줄이 나열된 목록보다 트리로 보는 것이 자연스럽습니다.

```text
main
  section
    h1
      "렌더링"
    p
      "브라우저가 화면을 만드는 과정"
```

브라우저는 HTML을 파싱하면서 이런 부모-자식 관계를 만들어갑니다.

## 브라우저는 HTML을 위에서 아래로 읽는다

브라우저는 HTML을 보통 위에서 아래로 읽습니다.

```html
<html>
  <head>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1>제목</h1>
    <script src="app.js"></script>
    <p>본문</p>
  </body>
</html>
```

읽는 도중 CSS 파일이 필요하다는 사실을 발견하면 CSS를 가져옵니다.

읽는 도중 script를 만나면 JavaScript 실행이 파싱에 영향을 줄 수 있습니다. 예를 들어 JavaScript가 `document.write`처럼 문서에 내용을 추가할 수도 있기 때문입니다.

그래서 브라우저 렌더링을 이해할 때는 HTML, CSS, JavaScript가 완전히 독립적으로 처리되지 않는다는 점도 중요합니다.

## 잘못된 HTML도 어느 정도 복구한다

브라우저는 실제 웹의 HTML이 항상 완벽하지 않다는 것을 알고 있습니다.

예를 들어 이런 HTML이 있다고 해봅시다.

```html
<ul>
  <li>첫 번째
  <li>두 번째
</ul>
```

`li` 종료 태그가 빠졌지만, 브라우저는 HTML 규칙에 따라 어느 정도 구조를 복구합니다.

```text
ul
  li
    "첫 번째"
  li
    "두 번째"
```

이런 복구는 브라우저가 HTML 파서를 갖고 있기 때문에 가능합니다.

## 핵심 정리

HTML 파싱은 단순히 태그를 찾는 일이 아닙니다.

브라우저가 HTML 문자열을 읽고, 요소와 텍스트를 구분하고, 부모-자식 관계를 만들고, 최종적으로 DOM을 만들 준비를 하는 과정입니다.
