# step2 — HTML이 DOM이 되기까지

### 정리
DOM을 흐릿하게 알면 흔히 이런 오해가 생깁니다.

`"DOM = HTML 아닌가?"
"document.querySelector가 그냥 HTML 텍스트에서 찾는 건가?"
"DOM은 처음에 만들어지면 끝 아닌가?"`

step2는 이 셋을 깨고 — DOM이 **HTML과 다른 객체 구조**이고, **JavaScript가 다룰 수 있는 형태**이며, **언제든 변할 수 있다**는 감각을 박는 단계


HTML 파싱은 HTML 문자열을 브라우저가 이해할 수 있는 구조로 해석하는 과정
파싱의 결과는 트리 구조
HTML, CSS, JavaScript가 완전히 독립적으로 처리되지 않는다는 점도 중요


DOM은 HTML 문서를 브라우저와 JavaScript가 다룰 수 있는 객체 트리로 표현한 것
브라우저는 HTML을 DOM이라는 객체 구조로 바꿉니다. DOM은 객체다!



### 질문
- 파싱이란 무엇인가요?
=> HTML 문자열을 브라우저가 이해할 수 있는 구조로 해석하는 과정

- HTML이 트리 구조로 표현되기 좋은 이유는 무엇인가요?
=> HTML 문자열을 작성하는 형식자체가 트리 구조에 가까워서.

- 브라우저가 HTML을 위에서 아래로 읽는다는 사실은 왜 중요할까요?
=> HTML을 작성할 때 규칙성 있게 작성할 수 있어서.

- DOM은 HTML과 어떤 점이 다른가요?
=> DOM은 HTML 문자열을 파싱하여 만든 트리구조 객체임.

- JavaScript가 HTML 문자열 대신 DOM을 다루는 것이 왜 더 안전하고 편할까요?
=> DOM은 객체기때문에 문자열보다 안전하고 접근이 편함.

- DOM만으로는 화면을 완성할 수 없는 이유는 무엇인가요?
=> CSSOM, Layout, Paint를 모르기때문에.
