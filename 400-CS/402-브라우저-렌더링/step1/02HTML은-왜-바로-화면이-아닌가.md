# HTML은 왜 바로 화면이 아닌가

## 한 문장으로

HTML은 화면 그 자체가 아니라, 브라우저가 화면을 만들기 위해 읽는 문서 구조 설명서입니다.

## 왜 바로 픽셀을 찍을 수 없을까?

화면은 결국 픽셀의 모음입니다.

픽셀을 찍으려면 브라우저는 최소한 아래 정보를 알아야 합니다.

```text
무엇을 그릴까?
어디에 그릴까?
얼마나 크게 그릴까?
무슨 색으로 그릴까?
다른 요소와 겹치면 어떤 순서로 그릴까?
보이는 요소인가, 숨겨진 요소인가?
```

그런데 HTML만 보면 이 정보가 모두 들어있지 않습니다.

```html
<button>저장</button>
```

이 코드는 "저장이라는 글자가 들어간 button 요소가 있다"는 구조를 알려줍니다.

하지만 아래 질문의 답은 아직 부족합니다.

```text
버튼의 너비는 몇 px인가?
버튼의 높이는 몇 px인가?
버튼은 화면의 x=몇, y=몇 위치에 놓이는가?
배경색은 무엇인가?
글자색은 무엇인가?
브라우저 기본 버튼 스타일을 쓸 것인가?
CSS에서 따로 지정한 스타일이 있는가?
```

그래서 브라우저는 HTML을 바로 픽셀로 바꾸지 못합니다. 먼저 HTML을 해석하고, CSS와 합치고, 위치와 크기를 계산해야 합니다.

## HTML은 구조를 설명한다

아래 코드를 봅시다.

```html
<article>
  <h1>브라우저 렌더링</h1>
  <p>HTML은 문서 구조를 설명합니다.</p>
  <button>자세히 보기</button>
</article>
```

HTML이 알려주는 것은 이런 구조입니다.

```text
article
  h1
    "브라우저 렌더링"
  p
    "HTML은 문서 구조를 설명합니다."
  button
    "자세히 보기"
```

이 구조는 중요합니다. 하지만 아직 화면은 아닙니다.

화면이 되려면 추가 계산이 필요합니다.

```text
article은 어느 위치에 놓을까?
h1 글자 크기는 몇 px일까?
p는 h1 아래 몇 px 떨어져 있을까?
button의 기본 여백은 얼마일까?
전체 페이지 배경색은 무엇일까?
```

## 비유

HTML은 건물 설계도에 가깝습니다.

설계도에는 "여기에 방이 있고, 여기에 문이 있다"는 구조가 있습니다. 하지만 설계도 자체가 건물은 아닙니다.

실제 건물이 되려면 재료, 크기, 위치, 색, 시공 순서가 필요합니다.

브라우저도 비슷합니다.

```text
HTML: 어떤 요소들이 있는지 알려주는 설계도
CSS: 요소들이 어떻게 보여야 하는지 알려주는 스타일 규칙
Layout: 요소들의 실제 크기와 위치 계산
Paint: 계산된 결과를 화면에 그리기
```

## 핵심 정리

HTML을 바로 화면에 그릴 수 없는 이유는 HTML이 최종 픽셀 정보를 갖고 있지 않기 때문입니다.

브라우저는 HTML을 먼저 해석 가능한 구조로 바꾸고, CSS와 결합하고, 각 요소의 크기와 위치를 계산한 다음에야 화면에 그릴 수 있습니다.

## 스스로 답해보기

2. `<button>저장</button>`만 보고는 아직 알 수 없는 정보는 무엇인가요?
3. "HTML은 화면이 아니라 설명서다"라는 말을 내 말로 다시 써보세요.
