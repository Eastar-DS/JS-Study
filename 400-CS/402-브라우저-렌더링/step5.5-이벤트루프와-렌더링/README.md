# step5.5 이벤트 루프와 렌더링

이 폴더는 보강자료입니다.

`step5/JavaScript가-렌더링에-끼어드는-방식.md`까지 읽고 나서 아래 질문이 생기면 읽는 것을 추천합니다.

- JS가 메인 스레드를 오래 잡으면 왜 화면이 멈출까?
- 브라우저는 도대체 언제 화면을 다시 그릴까?
- 마이크로태스크 큐는 렌더링 직전에 실행될까, 후에 실행될까?
- `setTimeout`으로 애니메이션을 만들면 왜 끊길 수 있을까?
- `requestAnimationFrame`은 무엇이 다를까?

이 폴더는 이벤트 루프 공부와 브라우저 렌더링 공부를 하나로 묶기 위한 자료입니다.

이스타가 이미 공부한 이벤트 루프 지식 (Call Stack, Task Queue, Microtask Queue) 위에 "렌더 기회"라는 한 가지 개념을 더 얹습니다.

이 자료를 읽고 나면 가상 DOM, React Fiber 같은 개념을 공부할 때 "왜 작업을 작게 쪼개야 하는가"를 자기 언어로 설명할 수 있게 됩니다.
