# 이미지, 폰트와 Layout Shift

## 한 문장으로

이미지나 폰트가 늦게 로드되면서 요소의 크기나 글자 모양이 바뀌면, 이미 그려진 화면의 Layout이 다시 바뀌어 화면이 흔들릴 수 있습니다.

## 왜 실무에서 중요할까?

사용자는 페이지가 빨리 뜨는 것뿐 아니라 안정적으로 보이는 것도 중요하게 느낍니다.

아래 상황을 생각해봅시다.

```text
글을 읽으려는 순간 이미지가 늦게 로드된다.
이미지가 공간을 차지하면서 본문이 아래로 밀린다.
사용자가 누르려던 버튼 위치가 바뀐다.
```

이런 화면 흔들림은 사용자 경험을 크게 나쁘게 만듭니다.

## 이미지로 인한 Layout Shift

```html
<img src="banner.png" alt="배너">
<p>본문 내용</p>
```

브라우저가 이미지의 크기를 미리 모르면, 처음에는 이미지가 차지할 공간을 정확히 예약하지 못할 수 있습니다.

나중에 이미지가 로드되고 실제 크기를 알게 되면 본문이 아래로 밀릴 수 있습니다.

개선 방향:

```html
<img src="banner.png" alt="배너" width="800" height="400">
```

또는 CSS로 비율을 예약할 수 있습니다.

```css
.image-box {
  aspect-ratio: 2 / 1;
}
```

핵심은 이미지가 오기 전에도 브라우저가 공간을 미리 계산할 수 있게 해주는 것입니다.

## 폰트로 인한 화면 변화

웹폰트는 네트워크로 다운로드됩니다.

폰트가 늦게 도착하면 브라우저는 처음에 기본 폰트로 글자를 보여주다가, 웹폰트가 도착한 뒤 글자 모양을 바꿀 수 있습니다.

이때 글자 폭이나 높이가 달라지면 Layout이 바뀔 수 있습니다.

```text
기본 폰트로 먼저 표시
-> 웹폰트 다운로드 완료
-> 웹폰트로 교체
-> 글자 폭 변화
-> 줄바꿈이나 요소 높이 변화
-> Layout Shift 발생 가능
```

## FOIT와 FOUT

웹폰트 로딩에서 자주 나오는 용어입니다.

```text
FOIT: Flash of Invisible Text
- 폰트가 로드될 때까지 글자가 보이지 않는 현상

FOUT: Flash of Unstyled Text
- 기본 폰트로 먼저 보이다가 웹폰트로 바뀌는 현상
```

둘 중 무엇이 더 좋은지는 상황에 따라 다릅니다.

하지만 실무에서는 사용자가 내용을 빨리 읽을 수 있게 하고, 화면 흔들림을 줄이는 방향을 고민합니다.

## font-display

```css
@font-face {
  font-family: "MyFont";
  src: url("/fonts/my-font.woff2") format("woff2");
  font-display: swap;
}
```

`font-display`는 웹폰트가 로딩되는 동안 텍스트를 어떻게 보여줄지 제어합니다.

`swap`은 기본 폰트로 먼저 보여주고, 웹폰트가 준비되면 교체합니다.

## 렌더링과 연결

이미지와 폰트는 단순한 리소스가 아닙니다.

이 리소스들은 요소의 크기, 줄바꿈, 위치에 영향을 줄 수 있습니다.

그래서 늦게 로드되면 Layout이 다시 계산될 수 있고, 이미 보이던 화면이 움직일 수 있습니다.
