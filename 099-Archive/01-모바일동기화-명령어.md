# 모바일 동기화 명령어 정리

현재 기준:

- 원본 저장소: `/Users/eastar/Documents/github/JS-Study`
- iCloud Obsidian 미러 vault: `/Users/eastar/Library/Mobile Documents/iCloud~md~obsidian/Documents/JS-Study-mirror`
- launch agent 이름: `com.eastar.vault-sync`

## 한 번만 해두면 되는 것

### 수동 1회 동기화

```bash
chmod +x /Users/eastar/Documents/github/JS-Study/099-Archive/sync-vault.sh
/Users/eastar/Documents/github/JS-Study/099-Archive/sync-vault.sh
```

### 감시 스크립트 실행 권한

```bash
chmod +x /Users/eastar/Documents/github/JS-Study/099-Archive/watch-vault.sh
```

### launch agent 설치

```bash
mkdir -p "$HOME/Library/LaunchAgents"
cp "/Users/eastar/Documents/github/JS-Study/099-Archive/com.eastar.vault-sync.plist" "$HOME/Library/LaunchAgents/"
launchctl load "$HOME/Library/LaunchAgents/com.eastar.vault-sync.plist"
```

## 자주 쓰는 명령

### 에이전트 상태 확인

```bash
launchctl list | grep vault-sync
```

### 수동 동기화 1회 실행

```bash
/Users/eastar/Documents/github/JS-Study/099-Archive/sync-vault.sh
```

### 포그라운드 감시 테스트

```bash
/Users/eastar/Documents/github/JS-Study/099-Archive/watch-vault.sh
```

종료:

```bash
Ctrl + C
```

### launch agent 중지

```bash
launchctl unload "$HOME/Library/LaunchAgents/com.eastar.vault-sync.plist"
```

### launch agent 다시 시작

```bash
launchctl load "$HOME/Library/LaunchAgents/com.eastar.vault-sync.plist"
```

### 설정 파일 수정 후 재적용

```bash
launchctl unload "$HOME/Library/LaunchAgents/com.eastar.vault-sync.plist"
cp "/Users/eastar/Documents/github/JS-Study/099-Archive/com.eastar.vault-sync.plist" "$HOME/Library/LaunchAgents/"
launchctl load "$HOME/Library/LaunchAgents/com.eastar.vault-sync.plist"
```

## 문제 생기면 보는 로그

```bash
cat /tmp/com.eastar.vault-sync.out
cat /tmp/com.eastar.vault-sync.err
```

## 기억해둘 점

- 이 동기화는 단방향이다.
- 로컬 `JS-Study`가 원본이다.
- 모바일에서 수정한 내용은 다음 동기화 때 덮어써질 수 있다.
- `.git/`은 동기화하지 않는다.
- `.obsidian/workspace*`는 기기별 충돌 방지를 위해 제외한다.

## 리소스 감각

- 평소에는 대체로 가볍다.
- 파일 변경이 없으면 감시 프로세스만 대기한다.
- 파일을 많이 저장하거나 대량 변경이 있을 때 `rsync`가 잠깐 동작한다.
- `node_modules` 같은 큰 폴더는 제외해둬서 부담을 줄였다.
